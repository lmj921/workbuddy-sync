# 大小球模型修正方案 v2（含英超 + 准确率优先）

## 核心诊断

### 数据实证（338场EPL回测）

| U2.5赔率区间 | 场次 | 实际U2.5命中率 | 市场隐含 | 模型想打的EV |
|-------------|------|--------------|---------|-------------|
| ≥2.3（市场强看大）| 81 | **45.7%** | ~43% | +5~25%（幻觉） |
| ≥2.0（市场看大）| 215 | **43.3%** | ~50% | +3~18%（幻觉） |
| <2.0（市场看小）| 123 | **50.4%** | >50% | 负EV不推 |

**结论**：
1. 当市场看大球(U2.5赔率≥2.0)时，实际U2.5命中率只有**43.3%** → 模型给出53%的概率是虚高10pp
2. 模型联赛均值(2.95球/场)高于实际数据(2.73)，但这个偏差让模型"系统性看小"
3. **真正的问题**：模型用全赛季均值(偏高的2.95)算出U2.5≈43%→本来合理。但校准器把它拉到53%→与市场逆行→实际只有43%→全灭

### 根本原因链条（英超+中超通用）

```
训练数据联赛均值(2.95) → Poisson给O2.5=56.6%/U2.5=43.4% (✅合理)
      ↓
校准器(Isotonic) → U2.5从43%推到46-48% (+3~5pp) (⚠️方向有问题)
      ↓  
脑算/伤停调整 → λ减小 → U2.5进一步推到50-55% (❌严重偏高)
      ↓
EV计算: 53% × @2.30 = +22% EV (🚨虚幻正EV)
      ↓
推荐U2.5 → 实际命中率43% → 0/5全灭
```

### 为什么英超也有问题？
- 模型E0的联赛均值(1.63+1.32=2.95)来自**多赛季混合**（含22-23/23-24的高进球赛季）
- 但当前25-26赛季只有2.73球/场 → 模型偏高0.22球/场
- 校准器进一步在U2.5方向过度修正（isotonic在40-50%概率区往上推了5pp）
- 核心冲突：**模型因为数据偏老而高估进球(偏O2.5)→ 校准器又想纠正这个偏差往U2.5推→ 两个偏差叠加成了"自信但错误的U2.5信号"**

## 修正方案（4层，准确率优先）

### 🎯 核心理念变化：EV优先 → 准确率优先

```
旧逻辑：EV>3% → Kelly仓位 → 下注
新逻辑：raw_prob>55% + EV>8% + 市场效率折扣后EV仍>0% → 下注
```

---

### Layer 1: O/U市场效率折扣（最高优先级）

**改动文件**: `src/decision.py`

```python
# 新增：O/U市场效率因子
# 回测证明：当模型说U2.5有43-55%概率时，实际命中只有43%
# 即模型在O/U市场系统性高估自己的准确性
# 解决：对O/U市场的model_prob进行"市场效率折扣"

OU_MARKET_EFFICIENCY = 0.85  # O/U市场庄家效率系数
# 含义：在O/U市场，模型只有85%的信心应该来自自身，15%应尊重市场

def adjust_ou_prob(model_prob, implied_prob, market_type):
    """对O/U市场概率做市场效率调整"""
    if market_type not in ('over_2_5', 'under_2_5', 'over_3_5', 'under_3_5'):
        return model_prob
    # 向市场概率收缩
    adjusted = OU_MARKET_EFFICIENCY * model_prob + (1 - OU_MARKET_EFFICIENCY) * implied_prob
    return adjusted
```

**效果预测**：
- Crystal Palace case: 53.2% × 0.85 + 38.5% × 0.15 = **51.0%** → EV: 51.0%×2.32-1 = +18.3%（仍正但大幅缩水）
- 需要配合Layer 2的阈值才能真正过滤

### Layer 2: O/U市场硬规则（准确率优先）

**改动文件**: `src/decision.py`

```python
# O/U市场专用过滤规则
MIN_EV_OU = 0.08            # O/U市场EV门槛从3%提高到8%
MIN_PROB_OU = 0.50          # O/U市场概率下限50%（adjusted后）
MAX_MARKET_DIVERGENCE = 0.15  # 模型与市场概率最大允许偏差15pp

# 新增硬规则：
# Rule A: 调整后概率<50% → 直接弃（之前43-48%也推）
# Rule B: 模型概率 - 市场概率 > 15pp → 触发"过度自信警报"→仓位减半
# Rule C: U2.5赔率>2.3 + 模型U2.5<55% → 直接弃（市场看大且模型没强信号）
```

### Layer 3: 联赛均值季度滚动更新（英超+中超）

**改动文件**: `scripts/predict_single.py` + 新增 `config/league_params.json`

```python
# 问题：模型联赛均值(2.95)用的是多赛季混合，
# 但当前赛季只有2.73 → 模型基准就偏高
# 修正：动态联赛均值 = 0.6×训练集均值 + 0.4×当季实际(如果有)

LEAGUE_CURRENT_SEASON = {
    "E0": {"home_avg": 1.497, "away_avg": 1.237, "total": 2.734, "o25_rate": 0.525},
    "CSL": {"home_avg": 1.800, "away_avg": 1.350, "total": 3.150, "o25_rate": 0.593},
}

def dynamic_league_avg(trained_avg, current_season_avg, blend=0.4):
    """用当季数据修正训练集联赛均值"""
    return (1-blend) * trained_avg + blend * current_season_avg
```

**效果**：
- EPL λ_total: 2.95 → 0.6×2.95 + 0.4×2.73 = **2.86**（更贴近实际）
- CSL λ_total: 2.91 → 0.6×2.91 + 0.4×3.15 = **3.01**（修正方向正确：CSL实际更多球）

### Layer 4: 注单排序改为 "EV × raw_prob × accuracy_weight"（准确率优先）

**改动文件**: `src/decision.py` 的 `generate_plan()`

```python
# 旧排序：按EV降序 → EV高=排前面 → 但EV高不等于准确率高
# 新排序：Score = EV × raw_prob × market_confidence
# - raw_prob高 = 事件本身高概率 = 更容易命中
# - market_confidence = 1 - abs(model_prob - implied_prob)/model_prob
#   (模型越接近市场→定价越确定→越不可能翻车)

def bet_priority_score(ev, model_prob, implied_prob):
    """准确率优先的注单排序分"""
    confidence = 1.0 - abs(model_prob - implied_prob) / max(model_prob, 0.01)
    confidence = max(confidence, 0.3)  # 下限30%
    return ev * model_prob * confidence

# 效果：
# 高概率低EV注(如主胜60% EV+5%) → score = 0.05×0.60×0.85 = 0.0255
# 低概率高EV注(如U2.5 45% EV+20%) → score = 0.20×0.45×0.50 = 0.0450
# 
# 再加Layer2的Rule A(prob<50%弃注)后，低概率高EV的U2.5注直接被砍掉
```
