# TAPD Maple项目 - A1分组单子统计与开单方案

## 背景
- 项目: Maple, workspace_id = `20453876`
- 用户是A1工作室PM，需要统计A1分组（策划/美术/开发）未完成的需求和Bug
- 后续需要通过MCP代开子需求并自动排期

---

## 任务一：统计A1分组未完成单子

### 步骤 1：查询TAPD状态配置
- 调用 `tapd_fields_summary_get` + `tapd_field_detail_get` 获取story和bug的实际状态值列表
- 确定"未完成"对应的状态筛选条件

### 步骤 2：查询A1相关的需求（Story）
- **筛选方式**: 用标题前缀 `name=LIKE<Maple_A1>` 进行模糊匹配
- **状态筛选**: 排除已完成状态（根据步骤1结果确定，用TQL `status=NOT_EQ<closed>` 或类似语法）
- **分页查询**: limit=200，确保拉取全部
- **需要区分父需求和子需求**: 通过 `parent_id` 字段判断层级关系

### 步骤 3：查询A1相关的Bug
- **筛选方式**: 先查出所有未关闭Bug，然后按处理人筛选（需要先确认A1成员名单）
- 或者尝试 `title=LIKE<Maple_A1>` 看Bug是否也有标题前缀
- **备选方案**: 先拉一批数据，看实际的标题/处理人分布，再确定最佳筛选策略

### 步骤 4：分析汇总
- 按"策划/美术/开发"分组统计数量
- 输出：总数、按状态分布、按优先级分布、按处理人分布
- 标出超期/无排期的单子

## 任务二：后续开单流程

### 输入格式解析
用户提供格式：
```
【父需求标题】
父需求链接（用于获取parent_id）
处理人
1.子需求标题1 工时
2.子需求标题2 工时
...
```

### 开单步骤
1. **解析父需求链接** → 提取 parent_id (19位)
2. **创建子需求**: 调用 `stories_create`
   - workspace_id: 20453876
   - parent_id: 从链接解析
   - name: 子需求标题
   - owner: 处理人
   - effort: 工时（如"1d" → 换算为工时值）
3. **查询处理人排期**: 调用 `stories_get` 查询该处理人当前进行中/待开始的单子，找到最近的空闲时间段
4. **设置排期**:
   - 调用 `stories_update` 设置 `begin` 和 `due`
   - 按工时顺序排列（前一个子需求结束后下一个开始）
   - **联调单子特殊处理**: 需要确保客户端和服务器时间对齐
5. **反馈结果**: 输出每个子需求的ID、标题、处理人、预计开始/结束时间

### 排期逻辑
- 查询处理人当前所有未完成需求的 `begin` 和 `due`
- 找到最早可用的时间窗口
- 按子需求顺序依次安排（非联调单按顺序，联调单与其他端对齐）
- 联调类子需求：取客户端和服务器两方面最晚的完成时间作为联调开始时间

## 关键工具调用链

| 步骤 | 工具 | 用途 |
|------|------|------|
| 查状态配置 | `tapd_field_detail_get` | 获取story/bug状态字段可选值 |

| 查分类 | `story_category_get` | 了解分类结构 |
| 创建需求 | `stories_create` | 开子需求 |
| 更新需求 | `stories_update` | 设置begin/due排期 |
| 查排期 | `stories_get` | 查处理人现有排期 |
