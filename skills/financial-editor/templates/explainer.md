# 模板：一文读懂（Explainer）

**适用场景**：复杂事件/政策/数据发布后，需要给读者一篇"读完就懂"的解读。
**字数**：1500–3000 字。
**SLA**：突发事件 30 分钟内，政策解读 1 小时内。

---

## 骨架

```
【标题】主标：核心事实 / 副标：信号与展望

【导言｜3 行法】
- 结论句（What）
- 关键数字（So What）
- 信号/影响（Now What）

【Part 1｜发生了什么】
- 事件/数据的基本事实
- 关键数字、关键时点、关键主体

【Part 2｜为什么发生】（归因≥ 2 个维度）
小标题 1：[宏观/中观/微观/技术面/外部]
小标题 2：[另一个维度]
（多维度交叉归因 = 严肃风"厚度"的核心）

【Part 3｜市场在说什么】（多方观点≥ 2 个信源）
- 卖方机构 A：[观点 + 依据]
- 公募/买方 B：[观点 + 依据]
- 官方/监管表态：[原文引用]

【Part 4｜接下来看什么】（向前看 + 分时间层）
- 短期（1 周内）：观察 [具体指标]
- 中期（1 季度内）：关注 [触发条件]
- 长期（半年以上）：取决于 [变量]

【Part 5｜对普通人的影响】（翻译层，选配）
- 用白话把专业话题翻译成"对普通投资者/储户/购房者意味着什么"

【数据来源】
【合规声明】
```

## 必备元素检查

- [ ] 导言 3 行法（结论/数据/信号）
- [ ] ≥ 2 个归因维度
- [ ] ≥ 2 个独立信源
- [ ] 明确的"下一步看什么"
- [ ] 至少 1 张数据图 / 机制图
- [ ] 合规声明

## 写作要点

1. **不讲来龙去脉就讲解读**：用户已知道"发生了什么"，需要的是"为什么+会怎样"
2. **术语双层**：首次出现术语 → 白话注释（如 "MLF（中期借贷便利，央行向银行提供中长期资金的工具）"）
3. **拒绝"专家称"**：所有引述必须有具体机构/人名/时间
4. **拒绝个股推荐**：可讨论板块与行业，但不给买卖建议
5. **结尾强收口**：不要"让我们拭目以待"这种虚尾

## 常见结构变体

| 变体 | 适用场景 | 结构 |
|---|---|---|
| 5W1H 版 | 突发事件 | What/When/Where/Who/Why/How 逐一展开 |
| 因果链版 | 政策解读 | 政策 → 传导机制 → 影响范围 → 最终结果 |
| 对比版 | 数据发布 | 本期数据 vs 上期 / 预期 / 历史同期 |
| Q&A 版 | 公众疑惑类 | 5–8 个核心问题逐一作答 |
