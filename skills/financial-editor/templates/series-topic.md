# 模板：专题搭建（Series Topic）

**适用场景**：持续 3–30 天的重大议题（美联储加息周期、财报季、两会、行业大事件）。
**形态**：专题页 = 主稿 + 系列子稿 + Timeline + 数据面板

---

## 专题要素清单

```
【专题入口页】
├── 顶部 Banner（主视觉 + 一句话主旨）
├── 核心一段话导览（What it is / Why it matters / How to follow）
├── Timeline 模块（关键事件回顾 + 未来预告）
├── 主稿区（≤ 3 篇 deep-read / explainer）
├── 滚动区（实时快讯聚合）
├── 数据面板（核心指标实时更新）
├── 观点区（多方机构观点并列）
└── 延伸阅读（相关历史稿、背景资料）
```

## 专题策划 SOP

### 阶段 1｜立项（事件发生前 / 发生时）
回答 3 问：
1. 值不值做专题？（量化："影响人群规模"/"持续时间"/"传播价值"）
2. 做多深？（轻量=聚合页；重量=独立专题，含原创 deep-read ≥ 3 篇）
3. 做多久？（事件周期 + 3 天缓冲）

### 阶段 2｜骨架搭建
- 定主标题 + 核心一句话主旨
- 定 3–5 个子议题（对应 3–5 篇原创主稿）
- 定关键 Timeline 节点（已发生 + 未来可预期）
- 定数据面板的 5–10 个核心指标

### 阶段 3｜内容排产
| 内容类型 | 数量 | 负责形态 |
|---|---|---|
| 开篇深读 | 1 | deep-read |
| 子议题解读 | 3–5 | explainer |
| 实时快讯 | 持续 | breaking-news |
| 滚动播报 | 关键日 | rolling-live |
| 中段盘点 | 1–2 | recap-list |
| 收官深读 | 1 | deep-read |

### 阶段 4｜运营节奏
- **首日**：首发深读 + 专题页上线 + 首屏推荐位
- **每日**：至少 1 篇原创 + 快讯聚合 + Timeline 刷新
- **关键节点日**：提前 24h 预热 + 当日滚动播报 + 次日总结解读
- **收官**：发"回顾 + 长线影响"深读，归档专题页

## 专题标题范式

- `[主题]特别报道 | [时间周期]` —— 例："美联储加息周期·2025"
- `聚焦[事件] | [持续几天]大事记` —— 例："聚焦 A 股一季报 | 30 天看财报季"
- `[数字]看[主题]` —— 例："10 张图看懂 2025 一季度经济数据"

## 数据面板配置建议

根据专题类型选择 5–10 个核心指标，全部实时刷新：
- **宏观类**：主要指数、汇率、债券收益率、商品价格
- **行业类**：行业指数、龙头股价、资金流向、产业链关键价格
- **公司类**：股价、成交量、估值水平、分析师评级

## 专题下线检查

- [ ] 所有子稿链接可访问
- [ ] Timeline 收尾有"回顾"一条
- [ ] 专题页是否需要加"本专题到此收官"提示
- [ ] 沉淀归档至"历史专题"库
