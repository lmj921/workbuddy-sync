# Workflow：财经热点筛选（Hotspot Discovery）

> **用途**：当用户**没有给出明确热点**时（例如"有啥值得写的？""今天晨会选题""盘点一下"），从公开信源里找出**市场聚焦、多家媒体共同关注**的财经热点清单，打分排序交付。
>
> **核心取向**：**市场聚焦度 > 选题独家度**。本 workflow 的评分模型专为"选题发现"设计，与 `hotspot-evaluation.md` 的"写作前再评估"**不是同一套分数**，两者互补。
>
> **触发词**：有啥热点 / 今天有什么财经新闻 / 晨会选题 / 帮我找选题 / 盘点今天值得写的 / 今天市场在聊什么 / 盘前盘后选题 / 哪些事值得做稿 / 市场焦点。

---

## 🎯 设计原则（为什么不用原来的 5 维）

| 场景 | 关心的核心 | 评分取向 |
|---|---|---|
| **筛选阶段**（本 workflow） | 市场共识、全网在聊什么 | 媒体越多报 = 分越高 |
| **写作阶段**（hotspot-evaluation.md） | 同一条热点我们能解读得多不一样 | **解读差异化**（角度/数据/逻辑链/信源组合）= 加分 |
| **自检阶段**（quality-rubric.md） | 稿件质量 | 独特信源、独到分析 = ★ 加分项 |

**关键澄清**：
- 筛选阶段的"差异化"是**选题维度的独家**（只有我们一家在写）—— 这在本阶段**不是优势反而是劣势**，因为共识未形成
- 写作阶段的"差异化"是**解读维度的独到**（同一热点，我们的角度/数据/逻辑链/信源组合和别人不一样）—— 这才是真正的编辑功力

所以如果一条热点"全网都在写"，它在筛选阶段是**高分**（必做），到了写作阶段才去追问"面对这条共识热点，我们能做出什么不一样的拆解"。**顺序不能反，差异化的定义也不能搞混**。

## 📡 信源清单（5 层金字塔）

> Agent 在跑 discovery 时，**按层从上到下扫**。越靠上权重越高。

### 第一层 · 官方/监管（权威源，权重 ×1.5）
- 央行：货币政策、利率、MLF、逆回购、降准降息
- 证监会：新规、处罚、注册制动态、减持新规
- 银保监会 / 国家金融监督管理总局
- 统计局：CPI / PPI / GDP / 社融 / M2 / PMI
- 发改委：产业政策、项目审批
- 财政部：国债、财政数据
- 外管局：外储、跨境资金
- **上交所 / 深交所 / 北交所 / 港交所**：重大公告、ST/退市、上市审核

> 🔧 **取数方式**：`web_search` 官网 + 官方公众号最新披露；若用户接了 RSS 订阅更好。

### 第二层 · 主流财经媒体（权重 ×1.2）
同一事件被这一层越多家同步报道 → 共振度越高。

- **通讯社/电报**：财联社电报、新华财经、华尔街见闻快讯
- **市场媒体**：证券时报、中国证券报、上海证券报、21 世纪经济报道、第一财经、经济观察报
- **专业媒体**：财新、界面新闻、澎湃财经、格隆汇
- **券商早报**：中信、中金、华泰、国君、招商等晨会纪要（常见汇总源：Wind、同花顺、格隆汇晨报合集）

### 第三层 · 市场异动数据（权重 ×1.0，提供"量化证据"）
- **A 股**：涨跌停榜、板块涨跌幅、成交额异动、北向资金流向、龙虎榜、大宗交易、融资融券异动
- **港股**：南向资金、港股通异动、恒指/国指成分
- **美股**：三大指数、中概股 ADR 异动、美债收益率
- **商品/外汇**：原油、黄金、铜、美元指数、USDCNY
- **可转债 / ETF**：大额申赎、折溢价异常

> 🔧 **取数方式**：优先用 `neodata-financial-search`（自然语言即问即答），数据不够用 `finance-data-retrieval` 精确取。**本层是"异动信号"，不是最终选题；它的作用是让 Agent 主动发现"咦，这东西为啥在动"**，然后回第一、二层找到对应新闻事件。

### 第四层 · 社交与研究讨论（权重 ×0.8）
- 微博：财经热搜、话题榜
- 雪球：讨论量突然放大的股票/话题
- 公众号榜：头部财经号（自选哥、猫笔刀、远川等）当日头条
- 知乎/小红书：财经相关涨热问题
- 研报库（如对接过）：当日发布的卖方观点汇总

> 🔧 **取数方式**：`web_search` + 已有公众号/雪球爬取 skill（若用户装了）。若都没有 → 降级为"从前三层推断情绪度"。

### 第五层 · 海外联动与日程（权重 ×0.8）
- 海外事件日历：FOMC、非农、BLS、OPEC、欧央行、日央行
- 重大财报季：特别是 Mag7、台积电、科技股龙头
- 地缘/宏观：美国政治、中美经贸、局部冲突对大宗影响

> 🔧 **取数方式**：`web_search` "本周经济日历" + 财经媒体的日历板块。

## 🪤 三层漏斗（从原始信息 → 最终选题清单）

```
┌─────────────────────────────────────────────┐
│ 粗筛：抓近 N 小时全部候选（默认 24h）         │  可能 100+ 条
└──────────────────┬──────────────────────────┘
                   ↓
┌─────────────────────────────────────────────┐
│ 去重合并：同一事件多源表述 → 合并为 1 条 hotspot │  剩 15-30 条
│ 同时记录"被几家媒体报道" = 共振度基数             │
└──────────────────┬──────────────────────────┘
                   ↓
┌─────────────────────────────────────────────┐
│ 精筛打分：对每条跑"筛选 5 维"，取 Top N       │  输出 5-10 条
└─────────────────────────────────────────────┘
```

### ▪ 粗筛规则

**时间窗**：
- 盘中筛选：过去 24h
- 周报/周末晨会：过去 7 天
- 用户可自定义（"最近一周"、"本月以来"）

**类型限定**：
- 只保留**财经/市场/产业/宏观**相关
- 过滤纯娱乐、社会、体育、军事（除非对市场直接有影响，如地缘事件影响原油）

### ▪ 去重合并规则

**同一事件的判定**（任一满足即合并）：
1. 报道同一主体 + 同一动作 + 时间差 <12h（如"央行降准 0.5%" 被 N 家报道）
2. 指向同一数据发布（如"3 月 CPI 同比涨 0.1%"）
3. 同一板块/股票的同向集中异动（如"军工板块集体大涨 3%+"）

**合并后保留**：
- 事件主表述（取**最权威/最早**的一家）
- **共振清单**：哪几家媒体同步报道（权重加总）
- 首报时间 + 最新报道时间
- 关联数据链接（如有涨跌幅/资金流向）

### ▪ 精筛评分：筛选 5 维（0-10，共 50 分）

> ⚠️ 这是**筛选专用版**，和 hotspot-evaluation.md 的"写作前再评估"是两套。

#### 1️⃣ 重要性（Importance，0-10）
事件本身影响面。参考刻度与 hotspot-evaluation 一致：
- 10：全球/全国性政策、议息、黑天鹅
- 7：行业级事件
- 4：单家头部公司重大事项
- 1：一般性动态

#### 2️⃣ 时效性（Timeliness，0-10）
- 10：正在发生 / T+0
- 7：24h 内
- 4：3-7 天
- 1：一周以上

#### 3️⃣ **市场共振度（Resonance，0-10）** ⭐ 本 workflow 核心
**算法**：
```
共振度原始分 = Σ (报道该事件的媒体权重) × 报道数
  权重：第一层×1.5，第二层×1.2，第三层数据×1.0，第四层×0.8，第五层×0.8

归一化到 0-10：
  ≥8 家权威媒体同步 → 10
  5-7 家 → 7-9
  3-4 家 → 4-6
  1-2 家 → 1-3
  0 家（只有社交） → 0（基本不选）
```

**加分条件**：
- 证监会/央行/交易所**官方发声** → 自动 +2
- 上了微博财经热搜 TOP10 + 有主流媒体跟进 → +1
- 同时触发"市场异动"（板块大涨/大跌 > 3% 且有明确原因） → +1

#### 4️⃣ 关联度（Relevance，0-10）
与用户核心领域的契合度：
- A 股 / 港股 / 美股 / 基金 / 宏观 / 某板块
- 用户首次使用时**问一次偏好**，记住
- 契合度越高越高分

#### 5️⃣ 情绪度 / 传播势能（Buzz，0-10）
- 微博/雪球讨论量
- 有无情绪化关键词（"暴涨""崩盘""罕见""历史首次""创纪录"）
- 有无 KOL / 头部公众号当日头条命中

### ▪ 总分与建议动作

| 总分 | 建议 | 对应做法 |
|---|---|---|
| **40-50** | 🔥 **必做** | 立即快讯 + 一文读懂（explainer），配滚动播报，有条件做深度 |
| **30-39** | ✅ **推荐做** | 快讯 + 一文读懂 二选一或组合 |
| **20-29** | 🤔 **可做可不做** | 归入当日综合盘点或简讯 |
| **10-19** | ⏸️ **不建议单独做** | 放进"今日盘点"列表即可 |
| **< 10** | ❌ **劝退** | 不选 |

## 🔧 Agent 执行步骤（SOP）

### Step 1｜确认筛选参数（一次性问清，之后记住）

给用户**一个选择题**（不要问开放题）：

```
帮你从哪个维度找选题？
A. 全市场通用（默认，覆盖 A 股 + 港股 + 宏观）
B. 聚焦某板块（告诉我板块名）
C. 仅关注监管/政策动态
D. 仅关注市场异动（涨跌停、资金流向）
E. 盘点过去 1 周（周报场景）
```

默认 = A，时间窗 = 24h。

### Step 2｜按信源层级扫描（并发）

**并行调用**：
1. `web_search` —— 官方/主流媒体最新头条（第一、二层）
2. `neodata-financial-search` —— 今日板块异动 / 资金流向 / 涨跌停（第三层）
3. `web_search` —— 微博财经热搜 / 当日头部公众号榜（第四层）
4. `web_search` —— 本周经济日历 / 海外联动（第五层）

> ⚠️ **必须真调数据**，不能凭模型记忆编。每条选题必须给出"信源列表"。

### Step 3｜去重合并 + 共振度计算

1. 对抓到的所有候选做语义去重（同事件合并）
2. 合并后每条记录"被哪几家报道" → 算共振度原始分 → 归一化
3. 记录首报时间、最新报道时间、异动数据链

### Step 4｜5 维打分 + 排序

对合并后的每条跑筛选 5 维 → 降序排列 → 取 Top 5-10。

### Step 5｜输出榜单

按"📋 输出模板"格式交付，末尾给下一步指引。

### Step 6｜用户选中后，切换到 hotspot-evaluation.md

用户说"做第 1 条" → 把这条热点作为明确输入，转交给主 SOP 的 Step 2（hotspot-evaluation 的"写作前再评估"） → 正式进入出稿流程。

## 🛡️ 质量底线（Agent 必守）

1. **不编造信源**：每条选题必须能点出"被哪几家媒体报道"。若实际只有 1 家 → 共振度 ≤ 3，坦诚标注"暂未形成市场共识"。
2. **不编造数据**：板块涨跌幅/资金流向等数据必须从 `neodata-financial-search` / `finance-data-retrieval` 取，不能凭记忆。
3. **坦诚工具限制**：如果信源抓取受限（反爬、登录墙、敏感词拦截），要**明确告诉用户"这次只扫到 X 层，Y 层未覆盖"**，而不是假装全覆盖。
4. **动态时效**：每次筛选都带时间戳，过期不复用。
5. **选题独家 ≠ 加分**：筛选阶段"只有我们一家在写"通常意味着**共识未形成**，共振度低是正确的。注意区分两种"差异化"——筛选阶段讨论的是"选题独家"（劣势），写作阶段讨论的是"**解读独到**"（对同一共识热点做出不一样的角度/数据/逻辑链/信源组合拆解，这才是真正的编辑价值）。

## ⚡ 常见误判与纠偏

- **误把"热搜"当"热点"**：娱乐/社会热搜冲上榜不代表财经热点。要看是否**有市场/产业/宏观落点**。
- **误把"异动"当"事件"**：某板块无故大涨 → 先找原因新闻；找不到 → 标注"异动待观察"而非直接立选题。
- **误把"预告"当"已发生"**：经济日历里的"今晚 8:30 公布非农" ≠ 可以马上写，要等数据出来再算共振度。
- **被"小作文"误导**：传闻若无权威媒体跟进 → 共振度给低分 + 明确标注"未经证实"。
- **过度依赖社交热度**：微博热度高但官方+主流媒体都没跟 → 很可能是情绪泡沫或炒作。
- **把"差异化"误解为"找冷门选题"** ⭐：这是最容易犯的错。正解是"选题找共识，解读找独到"—— 全市场都在写的热点是必做项，差异化不是去挖冷门独家选题，而是在这条共识热点上做出**别人没做的角度/数据切片/逻辑推演/信源组合**。
