# 小白入门引导（3 步当好财经编辑）

> 本 workflow 是给**第一次使用本 skill 的用户**准备的引导脚本。
> 目标：让一个对财经只有基础了解的人，通过 3 轮对话，产出一篇合格稿件。

---

## 设计原则

- **不考用户**：不让用户回答"你要做 explainer 还是 deep-read"这种术语问题
- **给选项，不给问答题**：让用户在 A/B/C 里选，而不是自己编答案
- **默认兜底**：每个选项都有 default，用户说"随便"也能推进
- **即交即审**：每次出稿都附自检表，小白看完就知道哪里改

---

## 3 步引导脚本

### Step 1｜你手上有什么？（1 个选择题）

"开工之前，先告诉我你有什么："

```
A. 有一个新闻链接 / 公告 / 研报（我读一下）
B. 只有一句话，比如"美联储暂停加息"（我帮你查背景）
C. 我没有具体事件，让你帮我找今天值得写的选题
D. 想做某个板块/主题的综合稿，没有特定事件（我帮你找切入点）
E. 已经有草稿，想改写 / 润色 / 合规检查
```

**Agent 动作**：
- 选 A/B → 读材料 / 补背景 → 进入 Step 2
- 选 C → **转入 `workflows/hotspot-discovery.md`** 跑共振度榜单 → 用户从榜单挑中一条 → 进入 Step 2
- 选 D → 追问"哪个板块 / 时间周期"，再走 hotspot-discovery 的"板块定制版" → 进入 Step 2
- 选 E → 跳过 Step 2，直接进 Step 3 选渠道

---

### Step 2｜这条热点值不值得做？做成什么形式？（Agent 主导）

**Agent 动作**（用户不用开口）：
1. 调用 `workflows/hotspot-evaluation.md`，对热点做 5 维打分
2. 把分数 + 建议用**非专业语言**告诉用户
3. 推荐 1–2 个形态，每个形态用一句话解释

**给用户看的推荐格式**：

```
📊 我评估了一下这条热点（满分 50）：
- 重要性：8/10（影响整个 A 股）
- 时效性：10/10（刚发生）
- 独家性：3/10（全网都在写）
→ 总分 35/50

💡 推荐做 2 种稿子：
1. 【快讯】200 字的"抢先发"稿（5 分钟就能写完，适合抢时效）
2. 【一文读懂】1500 字的"讲明白"稿（30 分钟写，适合沉淀阅读）

你想要哪种？还是都要？
```

**用户回答后** → 进入 Step 3。

---

### Step 3｜发在哪儿？决定风格（1 个选择题）

"稿子最终要发在哪里？"

```
A. 严肃财经媒体 / 官网 / App（严肃风，克制专业）
B. 微信公众号（公众号风，像朋友聊天但有干货）
C. 今日头条 / 短视频账号（头条风，强钩子、短句）
D. 先给我一版严肃风，回头我自己改（默认）
```

**Agent 动作**：
- 按选择加载对应的 `_channels/<channel>/style.md`
- 组合：`templates/{形态}.md` + `_channels/{渠道}/style.md` + `_common/*`
- 出稿后附自检表（`quality-rubric.md` 7+1 条）

## 交付给用户的"标准长这样"

每次出稿，Agent 都按这个结构交付：

```markdown
# [稿件标题]

（正文）

---

📋 **出稿自检表**（本轮结果）
1. 标题主旨+角度+吸引力        ✅ 理由：
2. 数据捕捉及时+准确+信号      ✅ 理由：
3. 归因维度多元                ✅ 理由：
4. 逻辑清晰+观点鲜明+客观       ✅ 理由：
5. 向前看+多渠道观点            ⚠️ 理由：只引用了 1 家机构，建议再加 1 家
6. 多方平衡+无个股推荐嫌疑      ✅ 理由：
7. 简洁通俗+少术语堆砌          ✅ 理由：
★ 独特信息/数据源              — 理由：未涉及

综合结论：**可发**（建议补充 1 家机构观点后更稳）

📎 **下一步建议**
- 如果事件持续发酵，建议明天加一篇 [深度解读]
- 如果要发公众号，我可以直接帮你改成公众号风格
```

## 小白常见 5 问 FAQ（Agent 要能应对）

### Q1：我不知道这个热点重不重要怎么办？
A：告诉我热点是什么，我来打分。你只负责最后拍板。

### Q2：严肃风和公众号风到底差在哪？
A：一句话——严肃风像新闻联播，公众号风像朋友群里发消息。
细节见 `_channels/internal/style.md` vs `_channels/wechat/style.md`。

### Q3：我担心写出来违规怎么办？
A：每篇出稿前我都会过 `compliance-checklist.md`，有任何红线嫌疑我会直接停稿并告诉你原因。你不用自己背禁词表。

### Q4：数据我不会找怎么办？
A：告诉我需要什么数据（比如"GDP""某股票一季报"），本 skill 会自动调用金融数据检索工具去查。

### Q5：如果我只是想学，不发稿，能不能让你教我？
A：可以。我会在交稿时附"这里为什么这么写"的注释版，把每一段对应的规则都讲清楚。

## Agent 行为硬约束

1. **不问术语**：绝不问"你要 breaking-news 还是 explainer"，只给选项。
2. **先交稿再问**：如果用户已经表达清楚，**直接出稿**，自检表随稿附上。让用户看到东西，比让用户选选项更友好。
3. **自检不达标就修**：7 条基础项任何一条 ❌，必须自己先改一轮再交稿。
4. **劝退也是服务**：评估分低于 10 分的热点，诚实告诉用户"不建议做"，别为了凑交付硬写。
5. **结尾永远带下一步**：每篇稿子末尾给 1–2 个"如果要继续，下一步可以这样"的建议。
