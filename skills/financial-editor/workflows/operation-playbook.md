# Workflow：运营动作矩阵（热点 → 组合拳）

> **定位**：资深编辑 vs 搬运工的分水岭。
> **回答的问题**：这条热点值得做成一次**运营事件**吗？如果值得，配什么**动作组合**？
> **触发时机**：`hotspot-evaluation.md` 打完 5 维分之后，`form-routing.md` 落单篇形态之前。
>
> **触发词**：运营动作、动作矩阵、组合拳、专题策划、一文看懂、一图看懂、深度解读、滚动播报、编辑部大脑、热点怎么打、这条热点怎么运营、排一套方案、出一套组合拳。

---

## 🎯 核心理念

**写作视角** vs **运营视角** —— 这是本 workflow 存在的根本原因：

| 维度 | 写作视角（form-routing） | 运营视角（本文件） |
|---|---|---|
| 问题 | 这条写成什么形态 | 这条要不要做一个**组合战役**，配什么动作组合 |
| 输出 | 1 个稿件类型 | 一个**动作矩阵**（可能 3–5 个动作打组合拳） |
| 决策层级 | 编辑个人 | 编辑部 / 运营总监 |
| 典型场景 | 快讯、速评 | 专题、一文看懂、一图看懂、深度复盘 |

**一句话原则**：
> **小事件做单篇，大事件排组合。** 看到一个大热点能不能立刻在脑子里排出一套"T+0 / T+半天 / T+1 / T+3"的节奏，是资深编辑的核心功力。

---

## 📦 7 种运营动作（清单）

每个动作都是一个独立的运营交付物，可以单做，也可以组合。

| 编号 | 动作 | 英文 key | 适用场景 | 核心价值 | 产出形态 | 生产周期 |
|---|---|---|---|---|---|---|
| 1 | **快讯** | `breaking-news` | 任何突发、抢时效 | 信息首发、占位 | 一句话 + 3 要点，300–500 字 | T+5–30min |
| 2 | **滚动播报** | `rolling-live` | 持续发酵事件（发布会/听证会/极端行情） | 持续陪伴读者 | 实时更新长帖，Timeline 结构 | T+0 持续到事件落幕 |
| 3 | **一文看懂 / 一文读懂** | `explainer` | 复杂政策、专业事件（需要背景铺陈） | 降低理解门槛 | 长图文 1500–3000 字，5–8 分钟阅读 | T+0.5–1 天 |
| 4 | **一图看懂** | `infographic` | 数据密集、结构对比、流程类 | 快速传播、降低阅读成本 | 信息长图 / 卡片图，1 张图讲完 | T+0.5–1 天 |
| 5 | **专题页 / 专栏** | `series-topic` | 超级事件（议息、两会、超大 IPO、黑天鹅） | 流量聚合、多稿集结 | 聚合落地页 + 多篇子稿 | 1–7 天持续 |
| 6 | **深度解读 / 复盘** | `deep-read` | 事件发酵 48h 后有新信息/新数据 | 沉淀价值、SEO 长尾 | 2000–5000 字深度长文 | T+1–3 天 |
| 7 | **数据可视化 / 交互页** | `data-viz` | 长期追踪类（利率曲线、板块轮动、基金持仓） | 持续复用、品牌沉淀 | H5 / 看板 / 交互图 | 1 周+ |

> 注：`breaking-news` / `rolling-live` / `explainer` / `series-topic` / `deep-read` 等也对应 `templates/` 下的写作骨架；本 workflow 负责**决定要不要出这些动作**，`form-routing.md` 负责**决定每个动作落笔的具体形态**。

## 🧭 3 轴决策模型

一条热点该配多少动作，不是凭感觉排，而是**跑 3 个判断轴**然后叉乘。

### 轴 1｜事件能量（沿用 evaluation 总分）

| evaluation 总分 | 级别 | 动作预算 |
|---|---|---|
| **≥ 40/50** | 超级事件 | 5+ 动作组合拳（必含专题页） |
| **30–39** | 重点事件 | 3–4 动作组合 |
| **20–29** | 标准事件 | 1–2 动作 |
| **< 20** | 常规事件 | 单篇（快讯/一文读懂 二选一即可） |

### 轴 2｜事件生命周期（这事能"活多久"）

| 生命周期 | 典型时长 | 示例 | 动作偏好 |
|---|---|---|---|
| **闪电型** | < 6 小时 | 某公司临时公告、盘中异动澄清 | 快讯 + 速评，**不做专题和深度** |
| **持续型** | 1–3 天 | 财报发布、单次政策发布 | 快讯 + 一文看懂 + 一图看懂 + 深度复盘 |
| **长周期型** | 1 周 – 1 月+ | 议息周期、地缘冲突、大 IPO、两会 | 专题页 + 滚动播报 + 持续追踪 + 数据可视化 |

### 轴 3｜信息复杂度（决定形态偏好）

| 复杂度 | 特征 | 优选动作 |
|---|---|---|
| **低** | 单一事件、单一数据点 | 快讯、一图看懂 |
| **中** | 多数据关联、需要背景 | 一文看懂、深度解读 |
| **高 + 分歧大** | 政策模糊、市场分歧、多维博弈 | 专题页 + 深度解读 + 一图看懂（做派系对比）|

### 3 轴叉乘 → 动作组合包

**组合包模板**（高频 6 种，覆盖 80% 场景）：

#### 📦 组合 A｜超级事件 · 长周期（总分 ≥40 + 长周期 + 高复杂度）
```
T+5min   → 快讯（首发占位）
T+30min  → 一文看懂（降低理解门槛）
T+2h     → 一图看懂（数据对比可视化）
T+6h     → 开建专题页（集合所有子稿）
T+24h    → 深度解读（沉淀价值）
持续      → 滚动播报 + 数据可视化看板
```
**典型**：美联储议息周（决议 + 鲍威尔讲话 + 分歧）、两会、某巨头黑天鹅（如银行挤兑）

#### 📦 组合 B｜重点事件 · 持续型（总分 30–39 + 持续型 + 中复杂度）
```
T+5min   → 快讯
T+30min  → 一文看懂
T+4h     → 一图看懂（若数据密集）
T+24h    → 深度解读
```
**典型**：央行降准、某头部公司财报、单次政策发布

#### 📦 组合 C｜数据发布型（宏观/财报类，总分 25–35）
```
T+0       → 快讯（数据速报 + 3 条核心要点）
T+60min   → 数据稿（结构化拆解表格）
T+4h      → 一图看懂（同比/环比/历史对比可视化）
T+次日    → 深度解读（若数据超预期明显）
```
**典型**：GDP / CPI / 社融、头部公司单季财报

#### 📦 组合 D｜异动/突发 · 闪电型（总分 20–30 + 闪电型）
```
T+5min   → 快讯
T+30min  → 速评型一文看懂（500–1000 字，讲清原因 + 影响）
(不做深度和专题)
```
**典型**：盘中突发异动、公司临时公告、短期传闻澄清

#### 📦 组合 E｜标准事件 · 单品（总分 20–29）
```
T+30min  → 一文看懂（独立单品，无需抢时效）
或
T+半日   → 一图看懂（若以数据对比为主）
```
**典型**：行业月报、监管例行答记者问、中等规模公司公告

#### 📦 组合 F｜常规事件（总分 < 20）
```
T+60min → 快讯 或 一文看懂，二选一
```
**典型**：常规公告、例行数据、行业小动态。**能不做就不做，产能优先给大事件。**

## 🎬 运营节奏模板（时间线视角）

把组合包拉通成"**一天/一周的事件时间线**"，这是真正的编辑部排兵布阵。

### 📅 一天型（组合 B / C / D 常见）
```
08:00 经济日历预热（若有盘前预告）
      │
事件发生 → T+5min  快讯
      │    T+30min 一文看懂
      │    T+2h    一图看懂
      │    T+6h    深度解读初稿
      │    T+24h   深度解读发布 + 盘面总结
```

### 📅 一周型（组合 A 常见）
```
D0  事件发生 → 快讯 + 一文看懂 + 一图看懂 + 专题页搭建
D1  滚动播报持续 + 深度解读
D2  专家观点聚合稿 / 市场反应复盘
D3–D5 数据可视化看板上线 + 持续追踪稿
D7  一周综述 / 一图看懂本周全景
```

### 📅 长周期型（组合 A 的特大版本，如议息周期、两会）
```
W-1  预热：一文看懂"本次值得关注 5 个看点"
W0   事件周：每日快讯 + 滚动播报 + 专题页
W+1  余波：深度复盘 + 数据可视化长期化
```

## 🔧 决策输出模板（Agent 每次按此输出）

```
运营动作矩阵：[事件名称]

【输入】
- evaluation 总分：XX/50
- 生命周期判断：闪电型 / 持续型 / 长周期型
- 信息复杂度：低 / 中 / 高 + 分歧大

【推荐组合包】：组合 X（A/B/C/D/E/F）
【动作清单】（按时间排序）：

  ✅ T+5min   · 快讯          · 负责形态：breaking-news
  ✅ T+30min · 一文看懂      · 负责形态：explainer
  ✅ T+2h    · 一图看懂      · 负责形态：infographic（数据对比）
  ✅ T+24h   · 深度解读      · 负责形态：deep-read
  ⬜ 可选    · 专题页        · 触发条件：若事件 T+12h 仍在发酵
  ❌ 不做    · 滚动播报      · 原因：事件非持续型，无需陪伴

【渠道分布建议】
- 严肃风（官网/App）：快讯 + 深度解读
- 公众号：一文看懂 + 一图看懂
- 头条/短视频：一图看懂（拆卡片）

【产能估算】：X 篇稿 / X 张图 / X 人天

【下一步】→ 逐个动作调 form-routing.md 决定单篇形态与渠道
```

## 🧨 常见运营误判（务必回避）

1. **"大事件只出快讯"** → 最严重的产能浪费。大事件的长尾价值 80% 在深度和专题里，只出快讯等于把流量送给友商。
2. **"小事件上专题"** → 反向浪费。专题页的搭建成本至少 0.5 人天，若事件 3 天后无人再提，沉没成本巨大。
3. **"什么都做一图"** → 一图看懂需要**真正有对比价值的数据**，把纯文本硬塞进图不如写好正文。
4. **"专题 = 稿件堆砌"** → 专题页的灵魂是**结构化的阅读路径**（背景→进展→影响→展望），不是把相关稿件全扔进去。
5. **"滚动播报开了就忘了关"** → 滚动播报必须有**明确的截止条件**（事件落幕 / 超过 48h 无更新），否则烂尾比不开还糟。
6. **"深度稿硬赶 T+0"** → 深度稿的价值在"别人写完快讯你还在写快讯"时出现。真正的深度稿应该等到 T+24h 有新信息时才发力。
7. **"所有事件都按组合 A 打"** → 这是新手常见误区（"看起来更专业"）。**动作越多 ≠ 质量越高**，动作应该匹配事件能量。

## 🔗 与其他 workflow 的衔接

```
hotspot-discovery.md   （找到热点）
        ↓
hotspot-evaluation.md  （5 维评估，得到总分）
        ↓
operation-playbook.md  （本文件：决定做多少动作 + 什么动作）
        ↓
form-routing.md        （每个动作落到具体稿件形态）
        ↓
templates/ + _channels/ + _common/   （组装稿件）
        ↓
quality-rubric.md + compliance-checklist.md  （自检 + 合规）
```

**关键边界**：
- `operation-playbook.md` 只决定"**做什么动作、按什么节奏做**"，**不**负责单篇稿件怎么写。
- `form-routing.md` 只决定"**单篇稿件用什么模板**"，**不**负责运营节奏。
- 两者一上一下，一战略一战术。

## 🎯 给 Agent 的心法

1. **先问生命周期，再谈动作**：拿到热点先判断它"能活多久"，而不是上来就问"写什么形态"。生命周期错了，后面全错。
2. **产能是稀缺资源**：永远给大事件留产能，**小事件敢于不做**。单篇常规稿的机会成本就是一篇深度稿。
3. **组合拳有节奏**：T+5min / T+30min / T+2h / T+24h 是产业里被验证过的节奏，不要随意偏离。
4. **专题页要有主心骨**：每个专题页必须有**一个贯穿始终的叙事主线**（比如"美联储到底在纠结什么"），否则只是稿件列表。
5. **一图看懂 ≠ 配图**：一图看懂是**一张图就能让人看完整件事**，它是独立作品，不是正文的附属。
6. **主动提醒用户上组合拳**：当事件总分 ≥ 30 时，不要等用户开口，主动给出"我建议排一套 X 组合"的提案。**这是资深编辑最值钱的主动性。**
