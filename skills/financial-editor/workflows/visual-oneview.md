# 一图看懂工作流（visual-oneview.md）

> **用途**：当运营矩阵里出现"一图看懂"这个动作，或用户直接说"帮我做张一图看懂"，按本 SOP 走。
> **定位**：本 skill 的"图文双产"能力——**图 + 文配套交付**，图不是甩出去就完事。

---

## 0. 什么时候触发本工作流

Agent 命中以下任一条件，立即进入本 SOP：

| 触发信号 | 示例 |
|---|---|
| 用户明确说 | "一图看懂 XXX"、"来张一图看懂"、"帮我画一张图"、"出个图版" |
| `operation-playbook` 输出的组合包里含"一图看懂"动作 | 超级事件 / 政策解读 / 财报密集期，矩阵通常会排这条 |
| 用户说"配张图""要张图""再加张图" | 已有稿件的场景，补图增强传播力 |

---

## 1. 双路径决策（A 出图 / C 只给 prompt）

Agent 收到请求后，**第一步必须判断走哪条路径**：

```
用户原话里是否含："给我prompt" / "我自己画" / "ChatGPT 就绪" / "我用 Midjourney" / "只要文案"？
    ├─ 有 → 走 C 路径（只出 prompt，用户拿去自己画）
    └─ 无 → 走 A 路径（默认，直接调 image_gen 出图）
```

**两条路径交付物对比**：

| 路径 | 交付 | 使用场景 |
|---|---|---|
| **A · 直出图**（默认） | 主图（PNG）+ 3-5 条核心提要文本 + 公众号/头条的说明文字（朋友圈转发钩子） | 用户要"立刻能用"的成品 |
| **C · 只给 prompt** | `{topic}-prompt.md` 文档，含：ChatGPT 就绪版 prompt / Midjourney 就绪版 prompt / DALL·E 就绪版 prompt + 使用说明 | 用户想用自己的 GPT / MJ / Stable Diffusion 账号画 |

⭐ **A 路径默认附赠 prompt 副本（2026-04-23 新增）**：即使用户走 A 路径（直接出图），Agent 在"下一步建议"里**必须**同时给一句\"需要就绪版 prompt 带走？\"的选项。若用户要，立即产出 `{topic}-prompt就绪版.md`（ChatGPT / Midjourney / 极简三版 + 工具对照表）。这样用户既拿到成品图，又能把 prompt 存起来之后自己复用。

## 2. 事件类型自动归类（决定图风格）

Agent 扫描用户给的事件文字，按下表优先级（从上到下）匹配第一个命中项：

| 优先级 | 关键词触发 | 归类 | 图风格 | 典型事件 |
|---|---|---|---|---|
| 1 | 央行 / 证监会 / 两办 / 人大 / 国常会 / 新规 / 征求意见稿 / 政策 / 法规 / 办法 / 意见 | **政策解读** | 信息图（竖版长图、时间轴、条款卡片） | "降准 25BP 一图看懂"、"资本市场新国九条一图看懂" |
| 2 | 营收 / 利润 / 毛利率 / 同比 / 环比 / 财报 / 业绩 / Q1 / Q2 / 中报 / 年报 / 快报 | **财报数据** | 数据可视化（柱/线/环形图、对比卡、关键数字放大） | "贵州茅台 Q1 财报一图看懂"、"宁德时代年报拆解" |
| 3 | 赛道 / 行业 / 产业链 / 科普 / 是什么 / 为什么 / 全景图 / 图谱 | **赛道科普** | 卡通科普（人物对话、图标化、箭头流程） | "什么是固态电池一图看懂"、"AI 大模型产业链图谱" |
| 4 | 突发 / 暴跌 / 涨停 / 爆雷 / 宣布 / 紧急 / 停牌 / 跌停 / 触发熔断 | **突发事件** | 新闻现场（写实场景、红色预警色系） | "硅谷银行倒闭一图看懂"、"XX 股临时停牌" |
| 5 | 人物姓名 + 年份 / 公司名 + 历程 | **人物/公司专题**（新增） | 时间轴信息图（年份节点、关键事件卡） | "一图看懂马斯克这一年"、"一图看懂 OpenAI 五年" |
| 兜底 | 都不匹配 | **兜底 = 信息图风格** | 信息图 | 其他 |

**Agent 必须告诉用户**："根据事件性质，我自动选了 **XX 风格**（信息图/数据可视化/卡通科普/新闻现场/时间轴）。如果不合适，你可以直接说'换成 XX 风格'。"

## 3. Prompt 模板（7 字段结构）

所有图的 prompt 都按这个结构填，**保证 A/C 两路径复用同一份 prompt**：

```
[1 图类型] + [2 主题] + [3 核心数据/要点 3-5 条] + [4 视觉风格词] 
+ [5 色调（A 股红涨绿跌）] + [6 版式] + [7 文字层] + [8 排除项]
```

### 3.1 政策解读（信息图风格）模板

```
[1] Chinese financial infographic, vertical portrait (9:16), 
[2] 主题：{政策名称}核心解读
[3] 核心要点：
    - 关键节点 1：{日期 + 事件 + 数字}
    - 关键节点 2：{日期 + 事件 + 数字}
    - 关键节点 3：{日期 + 事件 + 数字}
    - 影响对象：{涉及主体}
    - 落地时间：{生效日期}
[4] 视觉风格：扁平化信息图（flat infographic），图标 + 时间轴 + 条款卡片，专业金融媒体视觉
[5] 色调：主色 #C0282D（中国红，庄重）+ 辅色 #F5E8D0（米黄）+ 数据高亮 #D4B96A（金）
[6] 版式：顶部大标题区 + 中部三段式信息卡 + 底部落地节点轴 + 右下角"资深财经编辑 · 一图看懂"水印位
[7] 文字层：所有文字必须是清晰可读的简体中文，标题字号最大，数字字号次大，说明文字最小
[8] 排除：no English text, no stock price arrows in green-rise-red-fall (西方惯例), no people portraits, no cartoon style
```

### 3.2 财报数据（数据可视化风格）模板

```
[1] Chinese financial data visualization poster, vertical portrait (9:16),
[2] 主题：{公司名} {期间}财报核心数据
[3] 核心数据：
    - 营收：{数值 + 同比/环比}
    - 净利润：{数值 + 同比/环比}
    - 毛利率：{数值 + 变化}
    - 业务结构：{Top 3 业务线占比}
    - 现金流：{数值 + 趋势}
[4] 视觉风格：专业数据可视化（data viz），柱状图 / 折线图 / 环形图组合，Bloomberg / WSJ 审美
[5] 色调：⭐⭐ A 股惯例——涨用红（#E60000），跌用绿（#00A050）；背景米白 #FAF8F3；辅助线灰 #D5D5D5
[6] 版式：顶部公司 logo 位 + 中部 4 宫格核心数据（大数字 + 同比小标签）+ 底部业务结构环形图
[7] 文字层：中文标题 + 阿拉伯数字 + 中文说明，数字必须放大强调
[8] 排除：no green-up-red-down (反 A 股惯例), no photo-realistic, no pie chart with >6 slices
```

### 3.3 赛道科普（卡通科普风格）模板

```
[1] Chinese finance cartoon explainer illustration, square (1:1) or vertical (4:5),
[2] 主题：{赛道/概念} 是什么 & 怎么赚钱
[3] 核心知识点：
    - 概念一句话：{定义}
    - 产业链上游：{主体}
    - 产业链中游：{主体}
    - 产业链下游：{主体}
    - 代表公司：{A / B / C}
[4] 视觉风格：友好卡通风（friendly cartoon），类似回形针 PaperClip / 半佛仙人的插图质感，人物圆润、图标化、箭头流程清晰
[5] 色调：低饱和暖色（米黄 #FAF3E7 + 浅蓝 #A8C8E0 + 点缀橙 #F5A742），避免强烈红绿，因为是科普不是行情
[6] 版式：顶部一句话定义（大字）+ 中部产业链横向流程图（上游→中游→下游）+ 底部"代表公司"3 张小卡片
[7] 文字层：中文为主，术语可保留英文缩写（如 AI / GPU / ETF），加注中文解释
[8] 排除：no realistic human faces, no complex chart, no stock ticker symbols
```

### 3.4 突发事件（新闻现场风格）模板

```
[1] Chinese breaking financial news visual, vertical portrait (9:16),
[2] 主题：{事件名称}
[3] 核心要点：
    - 发生时间：{日期 + 时间}
    - 涉事主体：{公司/机构/人物}
    - 冲击量级：{金额/涨跌幅/市值变化}
    - 官方/监管回应：{一句话}
    - 市场传导：{板块/指数反应}
[4] 视觉风格：新闻编辑部审美（newsroom aesthetic），模拟电视新闻直播底图感，强现场感但不夸张
[5] 色调：红色预警主调（#E60000）+ 深灰背景（#2A2A2A）+ 黄色高亮（#FFD700）标关键数字
[6] 版式：顶部"突发"红色标签 + 中部大事件标题 + 时间线式 5 点要点 + 底部市场反应数据条
[7] 文字层：中文标题 + 明确的时间戳（精确到时分）
[8] 排除：no sensational imagery, no blood / explosion / disaster visuals, no political figures
```

### 3.5 人物/公司专题（时间轴信息图）模板

```
[1] Chinese timeline infographic, vertical portrait (9:16) with extra height allowed,
[2] 主题：一图看懂 {人物/公司} 的 {时间跨度}
[3] 时间节点（按时间顺序 5-8 条）：
    - {年份 1}：{关键事件 + 一个数字}
    - {年份 2}：{关键事件 + 一个数字}
    - ...（省略）
[4] 视觉风格：时间轴信息图（timeline infographic），左右交错卡片布局，中间纵向时间线
[5] 色调：中性商务（深蓝 #1F3A5F + 金色高亮 #D4B96A + 米白底 #FAF8F3）
[6] 版式：顶部人物/公司 logo 大图位 + 中部纵向时间轴（左右交错事件卡）+ 底部"三句话总结"
[7] 文字层：年份放大强调，事件描述一句话以内
[8] 排除：no celebrity face recognition concerns（生成通用剪影或图标替代真人肖像）, no political content
```

## 4. 合规前置自检（出图前必跑，轻量版）

为了避免"慢"，**用简化版合规自检**——只过 3 个硬红线：

| 检查项 | 命中怎么办 |
|---|---|
| ① **煽动性文案**：prompt/文案里含"必涨/一夜暴富/稳赚/抄底/翻倍" | **自动删除或改写**这些词，不命中才出图 |
| ② **个股推荐暗示**：prompt 出现"买入 XX / 推荐 XX / 布局 XX" | **改成中性表达**（"关注 XX / XX 表现值得跟踪"） |
| ③ **敏感主体**：prompt 含政治人物、宗教符号、民族敏感话题 | **直接停止**出图，告知用户 |

**Agent 必须在出图前显式说一句**："已过合规前置自检：① 文案无煽动词 ② 无个股推荐暗示 ③ 无敏感主体。可以出图。"

完整合规（`compliance-checklist.md`）留在出完图之后的配套文案环节跑。

## 5. A 路径执行步骤（默认）

```
Step 1｜事件类型归类 → 告知用户选了什么风格（给一句话理由）
  │
Step 2｜填入 7 字段模板 → 生成结构化 prompt（对用户展示 prompt 前 2 行摘要即可，不必全展开）
  │
Step 3｜合规前置自检（§4）→ 过了才继续
  │
Step 4｜调 image_gen 工具出图
  │     - size: 竖版信息图用 "1024x1536"，正方形科普用 "1024x1024"
  │     - quality: 默认 "medium"，用户说"高质量"用 "high"
  │     - n: 默认 1 张，用户说"多出几张选"用 2-4
  │     - output_dir: 工作区 "generated-images/" 子目录
  │
Step 5｜并行生成配套文案：
  │     - 核心提要（3-5 条，用于公众号正文穿插 / 头条话题 / 朋友圈转发钩子）
  │     - 一句话图说（图片 caption，100 字内）
  │     - 转发钩子（公众号风 / 头条风，按用户渠道定）
  │
Step 6｜过 compliance-checklist.md 完整合规（与常规稿件相同）
  │
Step 7｜交付：图片路径 + 配套文案 + 合规结论 + 下一步建议
```

## 8. 常见问题

**Q1：用户说"图太丑了重画"怎么办？**
→ 追问是"风格不对"还是"细节不满意"。
- 风格不对 → 换一套 prompt 模板（比如政策解读换成数据可视化风），重跑
- 细节不满意 → 在原 prompt 末尾追加具体修正指令（"换成横版"、"把标题放大"、"换成蓝色主调"），重跑

**Q2：用户说"图和文案不匹配"怎么办？**
→ 以图为准重写文案，或以文案为准重画图——问用户想保哪个。

**Q3：用户要多张图（系列）怎么办？**
→ 按系列规划：每张图一个子主题，整体保持统一风格（同一套 prompt 模板，只换核心要点）。

**Q4：image_gen 工具生成失败怎么办？**
→ 先看是否是尺寸不合规（hunyuan-image 模型 width × height ≤ 1024²），改成 "1024x1024" 兜底。再失败，切 C 路径告诉用户"自动生图临时失败，已为你准备 prompt，用你的 ChatGPT 手动画一下"。

## 9. Agent 心法

1. **图不是装饰，是信息载体**：每张图必须有明确的"信息骨架"（5 条核心要点），没有骨架的图就是抠脚美工。
2. **A 股红涨绿跌不可错**：数据可视化场景，涨红跌绿是**硬规则**，prompt 里必须显式写，否则 AI 默认跟欧美惯例画错。
3. **图文必须配套**：只出图不给文案 = 偷懒。用户拿到图还要自己配文字，体验断层。
4. **合规前置轻量版**：全量合规放文案环节。但煽动词 / 个股推荐 / 敏感主体这三条必须前置，因为画出来再改成本太高。
5. **默认 A，但要有边界感**：用户有明确"我自己画"信号，立刻切 C，别硬塞图。
