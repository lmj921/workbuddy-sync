# Workflow：公众号成稿交付（WeChat Publish）

> **硬性原则 1**：本 skill **不自动推送**到公众号草稿箱或群发接口。发布动作必须由人类编辑在微信后台手动完成。
> **硬性原则 2**：默认产出**"可直接粘贴"的公众号成稿三件套**，由用户自己复制到公众号后台"新建图文" → "保存为草稿"。

---

## 🎯 为什么默认不自动推送

Agent 在个人电脑端无法稳定完成微信公众号草稿箱的自动写入，原因：

| 硬性门槛 | 说明 |
|---|---|
| ① 公众号类型 | 必须是**已认证的服务号/订阅号**，个人订阅号 API 权限受限 |
| ② 凭据暴露风险 | 需要 AppID + **AppSecret**，任何对话回显都是泄露事故 |
| ③ IP 白名单 | 调用 `/cgi-bin/draft/add` 必须在公众号后台加白名单，个人电脑公网 IP 通常是动态的，不稳定 |
| ④ 封面素材 | 需先走 `material/add_material` 上传换 `media_id`，不能直接贴 URL |
| ⑤ 时效收益 | 即便技术通了，用户仍需去后台人工复核 + 点发送，节省的时间 ≤ 复制粘贴 |

**结论**：复制粘贴路径**最快、最安全、最可控**——用户在后台扫一眼稿件、定封面、点发送，这本来就是"专业编辑的必要动作"，不是负担。

---

## 🚀 默认路径：产出粘贴版成稿三件套

当用户说"发公众号 / 推草稿箱 / 帮我上架 / 这篇稿子进公众号后台 / 发推送"，Agent 的标准动作：

### Step 0｜能力边界一句话交代（**不要跳过**）

用一句话告诉用户"我不自动推，给你粘贴即用的成稿"：

> 公众号草稿箱的自动写入有 AppSecret + IP 白名单的硬门槛，个人电脑端跑不稳。我给你产出**可直接粘贴**的三件套，你打开后台 `Ctrl+V` + 传封面 + 保存为草稿，**全程 1 分钟**。

**只说一次**，不问"要不要这样"——这是默认路径，用户不反对就往下走。只有用户明确要求"自动推"时才走路径 B（见下）。

### Step 1｜产出三件套（按顺序写入 artifact 目录）

| # | 文件 | 作用 | 关键要求 |
|---|---|---|---|
| 1 | `{topic}-wechat.md` | 正文 MD 版 + 发布元信息 | 含：标题、摘要 ≤120 字、作者、正文 Markdown、7+1 自检表、合规声明 |
| 2 | `{topic}-wechat.html` ⭐**主交付** | 公众号后台粘贴即成型的 HTML | 含完整样式：标题 / A股红涨绿跌表格 / 引用块 / 加粗 / 互动引导 / 灰底免责声明 |
| 3 | `{topic}-cover.md` | 封面图文案 + 操作手册 | 3 方案封面文案 + AI 生图 Prompt（中英双版本）+ 上稿 5 步保姆级 + 时效提醒 |

> ⚠️ **HTML 样式要求**：使用内联样式（`<div style="...">`），公众号后台对外链 CSS 不兼容；颜色遵循 A股红涨绿跌规范；段落/引用/表格/分割线一应俱全。

### Step 2｜交付时必说的 5 句话

1. **双击 HTML 用 Chrome 打开** → `Ctrl+A` → `Ctrl+C`
2. **公众号后台"写文章"** → 光标到正文区 → `Ctrl+V`
3. **标题/摘要**从 `.md` 文件复制（避免 HTML 粘贴时丢失）
4. **封面图**按 `cover.md` 的 Prompt 自己生图或用备选方案
5. **点"保存为草稿"**——之后在后台定时发送

### Step 3｜给时效提醒

特别是盘中/盘后的实时热点，必须提醒：

> 现在 HH:MM，建议 **HH:MM 前上草稿、HH:MM 定时发送**。如果 [关键数据] 变化，标题就要改。

## 🛠️ 备用路径：自动推送（仅在用户明确要求时）

只有用户明确说"**我不要复制粘贴，我要自动推**""帮我写自动推送脚本""我有 AppSecret"时，才切到自动推送。

### 安全红线

| 红线 | 处理 |
|---|---|
| appsecret 出现在对话 / 稿件 / 日志 / 回显 | **立即停止**，请用户把凭据写进本地配置文件 |
| 调用 `mass/sendall` 或任何群发接口 | **禁止**，代码层不提供 |
| 修改 `wechat.json` 之外的全局配置 | 禁止 |
| 未经用户授权就推送 | 禁止，每次推送前显式确认 |
| 稿件未过 quality-rubric + compliance-checklist | 推送前必须 blocked |

### 凭证存储

用户级配置目录 `~/.workbuddy/skills/financial-editor/config/wechat.json`（**不入库**，必须在 `.gitignore`）：
```json
{
  "appid": "wx******",
  "appsecret": "********",
  "access_token": "",
  "token_expires_at": 0
}
```

### 核心流程

```
① 读本地配置 → ② 刷 access_token → ③ 上传封面图（material/add_material）
  → ④ POST /cgi-bin/draft/add → ⑤ 返回 media_id → ⑥ 告知用户去后台人工确认
```

### API 要点

**刷 token**：
```
GET https://api.weixin.qq.com/cgi-bin/token
  ?grant_type=client_credential&appid={APPID}&secret={APPSECRET}
```
缓存到 `wechat.json`，有效期 7200s，提前 300s 刷新。

**上传封面**：
```
POST https://api.weixin.qq.com/cgi-bin/material/add_material
  ?access_token={TOKEN}&type=image
  Content-Type: multipart/form-data
  Body: media=@cover.jpg
→ 返回 thumb_media_id
```

**推送草稿**（到此为止，不调群发）：
```
POST https://api.weixin.qq.com/cgi-bin/draft/add?access_token={TOKEN}
Content-Type: application/json

{
  "articles": [{
    "title": "...",
    "author": "...",
    "digest": "...",
    "content": "<html>...</html>",
    "thumb_media_id": "{封面返回的 media_id}",
    "need_open_comment": 0,
    "only_fans_can_comment": 0
  }]
}
```

### 常见失败

| 错误码 | 原因 | 处理 |
|---|---|---|
| 40001 | token 失效 | 重新获取 token 重试 1 次 |
| 40164 | IP 不在白名单 | 提示用户去公众号后台加白名单后重试 |
| 45009 | 调用频率超限 | 等待 1 分钟重试 |
| 其他 | 通用错误 | 返回原始错误给用户，不自动重试 |

> 💡 **强烈建议**：即便走自动推送路径，Agent 也只负责**生成脚本 + 写操作手册**，让用户**在本地自己跑脚本**。AppSecret 永远不进入对话上下文。

## ✅ 交付前必过的三道闸

不论粘贴版还是自动推送，交付/推送**之前**必须全部通过：

1. **质量自检**：过 `_common/quality-rubric.md` 7+1 条，有 ❌ 先改
2. **合规审核**：过 `_common/compliance-checklist.md`，命中红线停稿
3. **用户可感知**：把自检表与合规结果**展示给用户**，不要静默通过

## 🧭 给 Agent 自己的心法

1. **不要替用户做发布决定**——Agent 的终点是"把稿子写对、递到用户面前"，发不发、什么时候发，是用户的专业判断。
2. **粘贴版是最优解**，不是降级方案——1 分钟的复制粘贴换来 100% 可控和零凭据风险，划算。
3. **HTML 样式是核心交付件**——内联样式 + A 股红涨绿跌 + 表格/引用块齐全，粘贴过去能"成型"是体验分水岭。
4. **时效必须说**——盘中/盘后的实时稿，晚 1 小时热度砍半，Agent 不提醒等于失职。
5. **AppSecret 永远不入对话**——即便用户主动贴过来，也要立刻让他从聊天记录里删掉，改写进本地 `wechat.json`。
