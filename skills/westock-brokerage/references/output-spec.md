# 通用输出规范

本文档定义了券商观点 Skill 所有模式的通用输出规范，各模式 Prompt 均需遵循以下规则。

## 一、文本格式规范

- 使用 Markdown 格式
- 表格、列表、emoji 增强可读性
- 关键信息用粗体标注（**加粗**）
- 分歧用 ⚠️ 标注
- **所有标点必须使用中文标点**（逗号，冒号：句号。分号；括号（））
  - 数字小数点 `.`、百分号 `%`、美元 `$` 除外
- 每个部分之间不加分割线

## 二、图表输出

尽可能多地使用图表增强表达：

### 文本柱状图 — 板块推荐排行

```
{板块名称}        {█数量对应柱状图} X家券商
{板块名称}        {█数量对应柱状图} X家券商
```

柱状图长度与数值成正比。

## 三、推送格式（企业微信适配）

- 标题简洁（≤20 字）
- 正文分段（每段 ≤200 字）
- 关键数字突出显示
- 尽可能多地使用表格输出
- 所有问题内容保持在企微最大一段消息限制字数内

## 四、禁止内容

| 禁止项 | 说明 |
|-------|------|
| ❌ 编造券商观点 | 所有观点必须来自原始文章 |
| ❌ 夸大共识程度 | 如实反映一致/分歧比例 |
| ❌ 选择性呈现 | 不得偏向某特定券商 |
| ❌ 过度解读差异 | 分歧需有原文依据 |
| ❌ 使用非今日数据 | 必须用最新获取的文章 |

## 五、第一人称表述规范

**券商观点必须第三人称 + 出处。**

| 允许 | 禁止 |
|-----|------|
| ✅ "中信证券认为……" | ❌ "我认为……" |
| ✅ "18 家券商看好……" | ❌ 合并多家观点时用"业内共识"之类虚指 |
| ✅ "据 XX 券商 x 月 x 日研报……" | ❌ 无法溯源的笼统表述 |

**原则：所有观点必须可溯源到具体券商和具体文章。**

## 六、输入不明确处理

| 用户输入情况 | 处理方式 |
|-------------|---------|
| 用户问"券商怎么看？"（未指定板块/个股） | 反问"您想了解哪个板块/个股的观点？" |
| 用户问"最近"（时间不明确） | 默认查询最近一周 |
| 用户指定了具体券商但问题不完整 | 返回大盘观点模式(模式2)结果，并追问具体想了解什么 |
