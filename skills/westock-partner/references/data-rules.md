# 数据规则与补充

本文档只包含**炒股搭子独有**的数据规则，不重复 `westock-data` / `westock-tool` 自身的命令文档。

- **命令语法、批量查询、代码格式** → 见 `westock-data` skill 的 SKILL.md
- **选股筛选语法、字段清单** → 见 `westock-tool` skill 的 SKILL.md
- **各高手的数据使用流程** → 见各 `expert-XX.md` 的「🔧 实战流程」段

## 一、降级策略：未安装 westock-data / westock-tool 时

如果用户没有安装这两个 skill，**使用联网搜索能力获取实时数据**作为替代：

- **行情数据** → 搜索"XX股票 实时行情""XX股票 PE PB"
- **财务数据** → 搜索"XX公司 最新财报""XX公司 营收 净利润"
- **资金流向** → 搜索"XX股票 资金流向""XX股票 北向资金"
- **新闻公告** → 搜索"XX公司 最新新闻""XX公司 公告"
- **一致预期** → 搜索"XX股票 机构评级""XX股票 目标价"
- **条件选股** → 搜索"低估值高分红股票""PE低于15的银行股"

联网搜索的数据同样**必须标注来源**，禁止编造。

## 二、第零步·宏观新闻查询（westock-data 未覆盖的能力）

SKILL.md 第零步要感知当下宏观环境，但 `westock-data` 的 `news` 命令只能查个股新闻，**无法获取宏观层面的新闻**（政策、地缘、央行、贸易战等）。需独立方案：

### 方式一：联网搜索（推荐）

直接使用联网搜索能力，搜集最新的宏观新闻、政策动态、地缘事件。关键词示例："今日A股宏观要闻""美联储最新利率决议""中美贸易最新动态"等。

### 方式二：免费 API（按优先级尝试）

**方案 A：财联社电报快讯**（国内政策 / A股宏观覆盖最强）

```bash
curl -s 'https://www.cls.cn/nodeapi/telegraphList?app=CailianpressWeb&os=web&rn=20' \
  -H 'Referer: https://www.cls.cn/telegraph'
# 返回 JSON → data.roll_data[].content 是新闻正文
```

**方案 B：华尔街见闻全球频道**（全球宏观 / 地缘 / 央行覆盖最强）

```bash
curl -s 'https://api-prod.wallstreetcn.com/apiv1/content/lives?channel=global-channel&client=pc&limit=20'
# 返回 JSON → data.items[].content_text 是新闻正文
# 可选频道：global-channel（全球，推荐）| a-stock-channel（A股）| us-stock-channel（美股）
```

**其他备选**（方案 A/B 失败时再尝试）：
- 东方财富 7×24：`https://np-weblist.eastmoney.com/comm/web/getFastNewsList?client=web&biz=web_724&fastColumn=102&pageSize=20`
- 金十数据：`https://www.jin10.com/flash_newest.js`（JS 变量 `newest`，跳过 `lock:true` 的 VIP 条目）
- 新浪财经滚动：`https://feed.mix.sina.com.cn/api/roll/get?pageid=153&lid=2509&num=20&page=1`

### ⚠️ 时效性校验（必做）

- 拿到数据后，**先检查返回新闻的日期** —— 必须是近 1-3 天内的。如果全是一周前甚至更早的，数据源异常，**不得使用**，改用联网搜索。
- **绝对禁止用模型训练知识"回忆"宏观新闻** —— 模型记忆里的宏观信息大概率是几个月甚至一年前的旧闻，用户会看到"去年的新闻被当作今天的"。
- 如果 API 和联网搜索都拿不到最新宏观新闻，直接跳过第零步并告知用户"暂时无法获取最新宏观信息"，**不要硬编**。

### 用法提示

拉最近 20 条快讯，快速扫描有没有重大系统性事件（关税/制裁/央行利率决议/地缘冲突等）。如果有，必须先点出来。每次讨论查 1-2 个 API 就够了，不要高频请求。

## 三、炒股搭子独有的数据约束

> "先查后写 / 交叉验证 / 时效标注 / 异常主动提"等核心数据规则见 SKILL.md 铁律 1 + 本文档第五段。本段只收录 SKILL.md 没讲的、炒股搭子独有的补充约束。

1. **市值分档标准（全局通用）**
   - <30 亿 = 迷你票 / 微盘股
   - 30-100 亿 = 小盘股
   - 100-300 亿 = 中盘股
   - 300-1000 亿 = 中大盘股
   - \>1000 亿 = 大盘股
   - 描述个股市值时严格按此分档，**300 亿是中大盘股，不是"小票"**

2. **批量查询更高效** —— 能一次查多只就不要多次查一只。批量出表格时，先对所有标的执行 `quote`（必要时加 `finance` / `consensus`），拿到返回值后再填表；**没查过的标的不能出现在表格里**。

3. **搜代码再查数据** —— 如果用户没给具体代码，先用 `search` 找到代码再查。

4. **选股用 westock-tool，查数据用 westock-data** —— 别搞混。

5. **货币单位必须正确标注** —— 港股返回港元 / 美股返回美元，禁止使用人民币符号。

## 四、westock-tool 在炒股搭子里的使用

### 触发时机

用户说"找几只/选股/筛标的/有没有符合 XX 条件的股票"时触发 westock-tool。例如：

- "帮我找几只低估值高分红的"
- "PE 低于 15、ROE 高于 15% 的银行股"
- "最近一周资金净流入最多的是哪些"
- "符合坤仔标准的大市值股票有没有"

### 配合流程

```
1. 从用户需求提取筛选条件 →
   条件从对应高手档案的「核心投资逻辑」和「🔧 实战流程」推导
2. 调用 westock-tool → 拿到候选列表
3. 用 westock-data 查候选的详细数据（quote / finance / consensus 等）
4. 让相关高手基于详细数据发表意见
```

### 关键原则

- **筛选条件从高手档案推导，不在本文档列举固定条件**
  - 例：做"坤仔式选股"，应先读 `expert-04-逆向派.md` 的「核心投资逻辑」和「🔧 实战流程」，再把他的选股逻辑（大市值 + 近期跌幅较大 + PE 历史低位）翻译成 westock-tool 的筛选条件
  - 同理：文仔式选股读 `expert-03`，钊仔式选股读 `expert-05`，以此类推
- ⚠️ **westock-tool 筛出来的是"候选池"，不是"推荐买入"** —— 高手会基于各自框架做二次过滤
- **具体的筛选语法、可用字段** → 见 `westock-tool` skill 的 SKILL.md

## 五、数据完整性验证

> 投资决策的数据错误后果严重（真金白银的损失），必须有**交叉验证 + 时效标注 + 异常警报**三道防线。

### 5.1 关键数据交叉验证

**触发条件**：只有涉及**买卖决策节点**（"能不能买""该不该卖""价位下得去吗""抄底合不合适"）时才必做，纯讨论/复盘不强制。

| 数据 | 主要来源 | 交叉验证方法 | 偏差容忍 |
|------|---------|------------|---------|
| **PE(TTM)** | `quote` 返回 `pe_ttm` | 当前股价 ÷ EPS(TTM)（EPS 从 `finance` 取近 4 季合计） | ±5% |
| **市值** | `quote` 返回 `mktcap` | 当前股价 × 总股本（总股本从 `profile` 取） | ±2% |
| **涨跌幅** | `quote` 返回 `pct_chg` | (当前价 − 昨收) ÷ 昨收 × 100% | ±0.1% |
| **EPS(TTM)** | 股价 ÷ PE 反算 | `finance` 里近 4 季 EPS 直接相加 | ±3% |
| **股息率** | `quote` 返回 `dv_ratio` | 近 12 个月分红总额 ÷ 当前股价 | ±0.2 pct |

**超出偏差容忍 → 数据源不一致，必须在输出里标注**：
- 例："PE 数据存在口径差异（quote 给 17.5 / 反算 18.9），按 `quote` 为准，差异可能来自 EPS 周期划分"

### 5.2 数据时效性标注

不同时段的数据处理规则：

| 查询时段 | 数据性质 | 处理 |
|---------|---------|------|
| **盘中**（9:30-11:30 / 13:00-15:00）| 实时 | 直接用 |
| **盘后 2 小时内** | 当日收盘后数据 | 直接用，但提一句"今日收盘数据" |
| **盘后 2 小时以上** | 隔段数据 | **必须在输出里标注"数据截止 YYYY-MM-DD HH:MM"** |
| **隔日/周末** | 隔日数据 | **必须标注截止时间**，并提醒"下一交易日开盘可能变动" |
| **盘中涨跌剧烈时（单日 >5%）** | 易变数据 | 若数据已经 30 分钟+，主动说"行情波动较大，建议用户实时复查" |

### 5.3 数据异常警报

查询返回出现以下情况时，**相关高手必须主动提及**，不能当数据没异常：

| 异常 | 可能原因 | 应对 |
|------|---------|------|
| **PE 为负或极端高**（>200）| 公司亏损 or 盈利极低 | 切换到 PB / PS 等其他估值工具；不能装作"估值便宜"或"估值贵"一笔带过 |
| **股价单日涨跌 >8%** | 重大事件（利好/利空/停牌复牌/商誉减值/业绩暴雷）| 查 `news` 看原因；没查到原因就明说"波动显著，原因未明" |
| **成交量异常**（日成交额 <1000 万 或 较均量 >3 倍）| 流动性差 or 异动 | 流动性差要警告"进出容易打穿价格"；异动要查消息面 |
| **财务关键项为 "-"** | 最新财报未披露 or 行业特殊 | 不能跳过，明说"季报数据未披露，基本面结论受限" |
| **一致预期机构数 <3 家** | 机构覆盖少 | 预期可信度打折，输出时标注"仅 X 家机构覆盖，预期偏差可能较大" |
| **PE Bands 历史区间不足 3 年** | 新股 / 业务结构变过 | 文仔的估值结论要注明"历史区间较短，仅供参考" |

### 5.4 数据降级链

当 `westock-data` 查询失败时的后备方案（按优先级）：

```
1. 重试 1 次（可能是临时网络问题）
2. 改用等价命令（如 quote 失败 → 试 batch quote）
3. 联网搜索实时行情（明确标注"数据来源：联网搜索，时效 XX"）
4. 都失败 → 诚实说"该数据暂时查不到"，不降级到"凭记忆估计"
```
